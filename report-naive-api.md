# Отчёт по использованию "хипстерского" решения

Видимо, не на 100% понял, что API должен содержать отдельно команды,
поэтому сделал только работу с уже непосредственно текстовым набором команд,
(при помощи функции `make`). Всё остальное полностью скрыто от пользователя.

Действительно, ООП решение как представлено в занятии приходит на ум первым
(я даже попросил LLM вывести api, и он сделал точно такое решение в виде api-класса,
хотя оригинал полностью функциональный). Ранее я не так сильно задумывался о
том, что в итоге практически всегда интерфейс API представлен в чём то
вроде объекта, поэтому даже затрудняюсь сходу представить функциональный stateless
API.
